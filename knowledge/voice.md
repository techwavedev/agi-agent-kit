# Voice & Writing Preferences

> Synced to Qdrant via `memory_manager.py knowledge-sync`. Retrieved semantically at dispatch time.

## Tone

- (e.g. Professional but approachable; avoid jargon unless the audience is technical)

## Writing Style

- Use active voice.
- Prefer short sentences and paragraphs.
- Use bullet points for lists of three or more items.

## Naming Conventions

- Variables and functions: camelCase (JS/TS) / snake_case (Python)
- Files: kebab-case
- Constants: UPPER_SNAKE_CASE

## Documentation

- Every public function needs a docstring / JSDoc comment.
- README should include quick-start, env vars, and development setup sections.
